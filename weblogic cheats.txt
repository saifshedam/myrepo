---- Update JVM in weblogic
Download JDK tar(archive) file, unzip , and create a symbolic link--ln -sf---symbolic link will save the hasstle to replace java path in each and every script or file.
Copy fies cacerts, java.security, java.policy from old version to new one in /security folder.
Edit JAVA_HOME into nodemanager.properties.
Edit JAVA_HOME into setDomainEnv.sh
Edit JAVA_HOME into setNMJavaHome.sh
export AVA_HOME ENV variable.

use below scripts to find java in all files,

find . -type f -name “*.sh” -exec grep -il JAVA_HOME {} \;
find . -type f -name “*.sh” -exec grep -il /usr/java/jdk1.8.0_112 {} \;
find . -type f -name “*.” -exec grep -il /usr/java/jdk1.8.0_112 {} \;
find . -type f -exec grep -il /usr/java/jdk1.8.0_112 {} \;
find . -type f -exec sed -i ‘s/ jdk1.8.0_112/ jdk1.8.0_121/g’ {} +



--- Reset Admin password for weblogic
Shut down the WebLogic domain.

$ $DOMAIN_HOME/bin/stopWebLogic.sh

Rename the data folder.

$ mv $DOMAIN_HOME/servers/AdminServer/data $DOMAIN_HOME/servers/AdminServer/data-old

Set the environment variables.

$ . $DOMAIN_HOME/bin/setDomainEnv.sh

Reset the password using the following command. Remember to substitute the appropriate username and password.

$ cd $DOMAIN_HOME/security
$ java weblogic.security.utils.AdminAccount <username> <password> .

Update the "$DOMAIN_HOME/servers/AdminServer/security/boot.properties" file with the new username and password. The file format is shown below.

username=<username>
password=<password>

tart the WebLogic domain.

$ $DOMAIN_HOME/bin/startWebLogic.sh
